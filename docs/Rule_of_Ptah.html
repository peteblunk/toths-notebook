The Rule of Ptah

Protocols for Collaborative Creation

Preamble: We honor Ptah (Craftsmanship) and Ma'at (Order) in all development. We build with intent, preserve what is sacred, and seek balance between creativity and structure.

THE BACKEND DOES NOT EXIST FOR ITS OWN GLORY. IT EXISTS SOLELY TO SUPPORT THE USER IN THEIR ETERNAL STRUGGLE TO WRESTLE MEANING FROM THE CHAOS OF EXISTENCE. THE SYSTEM MUST NEVER PUNISH THE USER FOR BEING HUMAN. WE CONVERT THE ANXIETY OF CHAOS (NUN) INTO THE PEACE OF ORDER (MA'AT).

  1. Part I: The Prime Directive (Preservation)
    1. Principle: "Before a Sacred Scroll is rewritten, its Current State must be revealed."
      1. When a file requires significant modification, the Assistant (Gemini) shall request the current file contents.
      2. The Designer (Pete) shall provide the current code from the repository ("The Territory").
      3. The Assistant shall draft new code that integrates with existing logic, strictly avoiding blind overwrites.
  2. Part II: The Edict of the Lotus (Mobile First)
    1. Principle: "The Scribe shall build for the palm before the desktop."
      1. All design and development shall prioritize the mobile user experience as the bedrock of creation.
      2. We seek to bring Ma'at (Order) to the user by delivering an unparalleled, holistic experience on the small screen.
      3. The glory of a feature is measured by its seamless function and aesthetic on a mobile device.
  3. Part III: The Ritual of the Branch (Workflow)
    1. Principle: "Cowboy Coding is Banished to the Abyss."
      1. Direct coding on the main branch is prohibited.
      2. A unique branch shall be initialized for every task using the format: git checkout -b feature/scope-name.
    2. Scope Definition:
      1. Before code is written, a clear Scope of Work must be defined and agreed upon.
      2. Development shall not deviate from the agreed Scope.
      3. Commits shall be performed only upon the completion of the Scope.
  4. Part IV: The Ritual of Verification (Quality Control)
    1. Principle: "Trust, but Verify."
      1. Debugging must be performed on both Desktop and Mobile interfaces.
      2. The command npm run build must successfully complete locally before merging.
    2. Production Deployment:
      1. Merging to main requires a successful local build.
      2. Pushing to Production (Vercel) requires a final round of debugging on the merged code.
  5. Part V: The Path of Wisdom (Education)
    1. Principle: "The Scribe learns until the end of days."
      1. Before initiating a branch, the Assistant shall select one technical concept relevant to the Scope.
      2. The Assistant shall provide a brief explanation of this concept to enhance the Designer's technical proficiency.
